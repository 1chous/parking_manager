# 🚗 Parking Manager API

**Parking Manager** — это высоконагруженный backend-сервис для автоматизированной городской системы контроля парковок. Приложение позволяет управлять парковочными зонами, отслеживать въезд/выезд транспортных средств, загружать карты-схемы стоянок и в реальном времени генерировать финансовую и статистическую аналитику.

## 💼 Бизнес-функционал

*   **Управление зонами парковки:** Создание, настройка тарифов, вместимости и загрузка графических карт-схем (`.jpg`, `.png`).
*   **Контроль сессий:** Регистрация заезда автомобилей (по номерному знаку), фиксация времени выезда и статуса оплаты. Защита от переполнения парковки.
*   **Аналитика:** Генерация сводных отчетов по каждой зоне (средняя заполняемость в процентах, средняя продолжительность стоянки, суммарная выручка).

## 🛠 Технический стек

*   **Язык и фреймворк:** Java 21, Spring Boot 3 (WebMVC, Data JPA)
*   **База данных:** PostgreSQL 16 + миграции Flyway
*   **Инфраструктура:** Docker, Docker Compose, Nginx Proxy Manager
*   **Нагрузочное тестирование:** Locust (Python)
*   **Документация:** OpenAPI 3.0 (Swagger)

## 🚀 Инженерные решения и оптимизации (Highload)

Проект спроектирован с учетом высоких нагрузок (Highload) и протестирован на удержание стабильного времени отклика (<200 мс).

### 1. Многостадийная сборка Docker (Multi-stage build)
Для минимизации веса и повышения безопасности итогового контейнера используется двухэтапный `Dockerfile`. Компиляция исходного кода происходит в тяжелом образе `gradle:9.5.1-jdk21-alpine`, после чего готовый артефакт (`.jar`) переносится в изолированный и минималистичный образ `eclipse-temurin:21-jre-noble`. Зависимости кэшируются на отдельном слое Docker.

### 2. Оптимизация слоя данных и устранение проблемы N+1
Тяжелый аналитический эндпоинт `/api/v1/analytics/summary` был полностью переписан. Вместо использования стандартных механизмов ORM (которые вызывали проблему N+1 и генерацию тысяч последовательных запросов при ленивой загрузке связанных сессий), логика вынесена на уровень СУБД.

Используется сложный `Native SQL` запрос с `LEFT JOIN` и математическими агрегациями (расчет `EPOCH`, `COALESCE`, вычисление процентов) прямо внутри PostgreSQL, что снизило время ответа эндпоинта в десятки раз.

### 3. Тюнинг PostgreSQL под стресс-тесты
Параметры базы данных в `docker-compose.yml` оптимизированы под аналитические нагрузки:
*   Выделены жесткие лимиты ресурсов (Memory, CPU).
*   Настроен кэш и рабочая память (`shared_buffers`, `work_mem`) для быстрых сортировок и группировок в ОЗУ.
*   Отключена синхронная запись (`fsync=off`, `synchronous_commit=off`) для максимизации RPS при массовом открытии парковочных сессий.

## 📖 Документация API

После запуска приложения, интерактивная документация Swagger UI доступна по адресу:
`http://localhost:8080/swagger-ui.html`
А также до запуска в файле `swagger.yml`

### Основные эндпоинты:
*   `GET /api/v1/zones` — получить список всех парковок.
*   `POST /api/v1/zones` — создать новую парковочную зону.
*   `PUT /api/v1/zones/{id}/image` — загрузить карту-схему (multipart/form-data).
*   `POST /api/v1/sessions` — начать новую парковочную сессию.
*   `PUT /api/v1/sessions/{id}` — завершить сессию / обновить статус оплаты.
*   `GET /api/v1/analytics/summary` — получить сводный аналитический отчет.

## ⚙️ Быстрый старт (Локальный запуск)

1. Клонируйте репозиторий:
   ```bash
   git clone ...
   cd parking-manager
   ```

2. Задайте пароль для базы данных в файле `.env` (или экспортируйте переменную):
   ```bash
   export DB_PASSWORD=your_secure_password
   ```

3. Запустите инфраструктуру и бэкенд через Docker Compose:
   ```bash
   docker compose up -d --build
   ```
   *Система автоматически поднимет PostgreSQL, выполнит миграции Flyway, запустит Nginx Proxy Manager (на портах 80, 443 и 81) и сам Spring Boot backend.*

## 📈 Нагрузочное тестирование (Locust)

В корне проекта находится скрипт `locustfile.py` для проведения стресс-тестирования системы. Он имитирует создание зон, загрузку бинарных файлов схем, генерацию номеров автомобилей и массовое открытие парковочных сессий.

Для запуска тестов:
1. Установите Locust: `pip install locust`
2. Запустите скрипт:
   ```bash
   locust -f locustfile.py --host=http://localhost:8080
   ```
3. Откройте веб-интерфейс `http://localhost:8089` для мониторинга метрик (RPS, Latency) в реальном времени.
